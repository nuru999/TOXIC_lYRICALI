Merge Conflict Demo

This section documents the Git merge conflict that was intentionally created and resolved during Phase 5 of the collaboration session.

What Is a Merge Conflict?

A merge conflict occurs when two branches make different changes to the same line of a file. Git cannot automatically decide which version to keep, so it pauses and asks the developer to resolve it manually.

How the Conflict Was Created

  1. Zakariya (260075) created branch feature/260075-conflict-demo from main.
  2. David (260056) created a separate branch from main and both edited the same line — the <title> tag in index.html.
  3. David's branch was merged into main first via Pull Request.
  4. Zakariya then ran git pull origin main — Git detected a conflict and halted the merge.

How the Conflict Was Resolved

  1. Opened index.html and located the conflict markers:
    <<<<<<< HEAD, =======, >>>>>>> origin/main
  2. Removed the conflict markers and kept the correct final content.
  3. Staged the resolved file: git add index.html
  4. Committed the resolution: git commit -m "fix: resolve merge conflict on page title"
  5. Pushed the branch and opened a final PR for David to review and merge.

Key Git Commands Used

  • git pull origin main — fetches remote changes and attempts to merge
  • git status — shows files with unresolved conflicts
  • git add index.html — marks the conflict as resolved
  • git commit -m "fix: resolve merge conflict on page title" — finalises the merge
  • git push origin feature/260075-conflict-demo — pushes resolved branch
